--- layout: post title: A Post-Award BIS Spring Release fix release is now published date: 2019-08-16 00:00:00 ---

 

A corrigendum hotfix to the Spring 2019 release of the PEPPOL Post-Award artefacts (BIS documents and validation artefacts) has now been published.

 

The corrigendum is mainly to adopt into the PEPPOL BIS 3.0 Billing the latest updates that have been made to the EN16931 validation artefacts and was published in June. Those mainly involve a correction to rules on due dates in Credit Notes and updates to code lists.

 

The updates to the code lists also affect other BIS specifications.

 

Details can be found in the release notes at:

 

http://docs.peppol.eu/poacc/billing/3.0/release-notes/
http://docs.peppol.eu/poacc/upgrade-3/releasenotes/

 

The release package and its release notes can be found at:

 

https://peppol.eu/downloads/post-award/

 

This release from OpenPEPPOL includes changes to the mandatory validation artefacts for all published PEPPOL BIS 3.0 documents as distributed by email to the OpenPEPPOL Post Award community on July 19th

 

The BIS 3.0 specifications in this release are mandatory from August 16th, 2019 12:00 CEST.

 

Validation artefacts are the tools that help parties and service providers to comply with the PEPPOL Transport Infrastructure Agreements (TIA) as they help to ensure that the contents of the exchanged document instances in the PEPPOL network are valid both from a syntactical and semantic point of view. These business rules check for the correctness of the coded values, co-occurrence constraints or calculations. It is therefore necessary to use them before submitting documents to the network, in order to ensure the overall quality of exchanged electronic documents, and thus the overall performance of the network.
